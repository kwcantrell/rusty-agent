# Summary

First-party material by Academic research community (published 2025-03-20). Key claims extracted below; the live document is the authority.

# Key claims

- Core autonomous system capabilities including planning, reasoning, and tool use must be systematically evaluated
- Foundational LLM capabilities (planning and tool use) form a primary evaluation dimension for agents
- Domain-specific benchmarks must address different agent types, particularly web and software engineering agents
- Generalist agent assessment requires evaluation approaches distinct from domain-specific methods
- Systematic analysis of core characteristics and dimensions of agent benchmarks is necessary
- Developer-focused evaluation frameworks are critical for supporting agent SDLC tooling
- The field is shifting toward more demanding and realistic evaluations
- Benchmarks must be continuously updated and refreshed to remain relevant
- Operational cost measurement is a significant gap in agent evaluation
- Safety assurance is an underexplored dimension in agent evaluation practices
- Robustness confirmation is a critical but underdeveloped evaluation requirement for agents
- Evaluation methods must balance detail and granularity with computational efficiency and scalability
