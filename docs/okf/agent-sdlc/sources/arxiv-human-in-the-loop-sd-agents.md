# Summary

First-party material by Academic research (accepted ICSE SEIP 2025) (published 2024-11-19). Key claims extracted below; the live document is the authority.

# Key claims

- Multi-agent LLM paradigms are used to automatically resolve software development tasks.
- Human-in-the-loop frameworks can be designed, implemented, and deployed to enable engineers to guide LLMs during planning and code generation phases.
- Existing agent research relies on historical benchmark datasets without incorporating human feedback throughout the development process.
- Multi-stage evaluation with actual engineers is required to assess real-world performance of agent systems.
- AI agent systems can be deployed in production platforms like JIRA for operational use by software engineers.
- Engineers perceive that human-in-the-loop AI agents can reduce overall development time and effort, particularly for initiating coding plans and writing straightforward code.
- Code quality remains an ongoing concern in agent-generated code, indicating a gap between perceived benefits and actual reliability.
- Integration of human oversight throughout the automated development workflow is critical for practical deployment of LLM-based agents.
