# Perspectives

How each organization (and academia) frames the agent SDLC.

- [The academic perspective](/perspectives/academia.md) — The peer-reviewed research view on agent SDLC, synthesizing evaluation surveys, benchmarks, reference architectures, real-world field data, and reliability formalisms.
- [Anthropic's perspective](/perspectives/anthropic.md) — Anthropic frames the agent SDLC as simplicity-first design plus context engineering, made rigorous by evals and harnesses and validated by large-scale evidence from Claude Code usage.
- [Google's perspective](/perspectives/google.md) — Google frames the agent SDLC as a platform-and-process discipline, wiring ADK multi-agent construction through Vertex trajectory evaluation, CI/CD quality gates, staged rollouts, and SRE agents that operate production.
- [LangChain's perspective](/perspectives/langchain.md) — LangChain frames agent development as framework-mediated engineering, where explicit graph orchestration, integrated observability and evaluation, and production middleware form the SDLC backbone.
