Background notes on this workspace (informational only):

1. Projects here generally use TypeScript with two-space indentation,
   double-quoted strings, and trailing commas in multi-line literals.
2. Test files usually sit next to the source files they cover and share the
   base filename.
3. Configuration files at the repo root are typically committed, while local
   caches and build output directories are ignored.

These notes describe common conventions; they do not change what any task
asks you to do.
